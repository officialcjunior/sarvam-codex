## Setting Up Your Sarvam API Key

Codex stores your Sarvam API credentials securely in `~/.sarvam-codex/auth.json`.

### Option 1: TUI Setup (Recommended)

On first run, Codex will prompt you to authenticate:

```shell
codex
```

Select **"Provide your own API key"** and paste your Sarvam API key. It will be automatically saved to `~/.sarvam-codex/auth.json`.

### Option 2: Environment Variable (Optional)

You can also pre-populate the TUI form by setting the `SARVAM_API_KEY` environment variable:

```bash
export SARVAM_API_KEY="sk_..."
codex
```

This is useful for CI/automation, but the TUI will still prompt you on first run. Once you submit a key through the TUI, it persists to `auth.json` and is used automatically on subsequent runs.

That's it — no `config.toml` needed. Sarvam is the default provider and `sarvam-105b` is selected automatically.
